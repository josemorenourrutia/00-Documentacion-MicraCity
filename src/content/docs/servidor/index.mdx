# Dependencias del Backend

Este proyecto utiliza **Node.js**, **TypeScript** y **Express**, y proporciona:

- APIs REST para la app móvil
- Gestión de usuarios y datos
- Subida y procesamiento de imágenes
- Conexión con la base de datos MySQL

A continuación se describen las principales dependencias utilizadas.

---

## Dependencias principales

<CardGrid>
  <Card title="Express" icon="logos:express">
    Express es un framework minimalista para **Node.js** que permite crear servidores web y APIs de forma sencilla.

    Se utiliza para:

    * Crear el servidor HTTP
    * Definir rutas (`routes`)
    * Procesar peticiones y respuestas

    Instalación:

    ```bash
    npm install express
    ```
  </Card>

  <Card title="Cors" icon="mdi:shield-cross">
    Middleware para habilitar **Cross-Origin Resource Sharing**, permitiendo que la app móvil o web se comunique con el backend.

    Esto permite que una aplicación frontend (por ejemplo una app en React Native o una web) pueda comunicarse con el servidor.

    Instalación:

    ```bash
    npm install cors
    ```
  </Card>

  <Card title="Dotenv" icon="mdi:circle-edit-outline">
    Carga variables de entorno desde un archivo .env, utilizado para credenciales y configuraciones sensibles.
    
    Se utiliza para guardar información sensible como:

    * Credenciales de base de datos
    * Claves secretas
    * Configuración del servidor

    Instalación:

    ```bash
    npm install dotenv
    ```

    Ejemplo:

    ```
    DB_HOST=localhost
    DB_USER=root
    DB_PASSWORD=1234
    ```
  </Card>

  <Card title="Multer" icon="mdi:upload">

    Es un middleware para manejar **subida de archivos** (como imágenes) desde la app al servidor en Express.

    Se utiliza en este proyecto para:

    * Subir imágenes
    * Procesar formularios con archivos
    * Guardar archivos en el servidor

    Instalación:

    ```bash
    npm install multer
    ```

  </Card>

  <Card title="Sharp" icon="mdi:image-edit">
    Es una librería para **procesamiento de imágenes de alto rendimiento** para procesar imágenes: redimensionar, comprimir y convertir formatos.

    Permite:

    * Redimensionar imágenes
    * Comprimirlas
    * Convertir formatos

    Instalación:

    ```bash
    npm install sharp
    ```
  </Card>

  <Card title="MySQL2" icon="logos:mysql">
    Es un driver que permite conectar Node.js con bases de datos **MySQL**.

    Se utiliza para ejecutar consultas SQL desde el servidor.

    Instalación:

    ```bash
    npm install mysql2
    ```

  </Card>

  <Card title="Sequelize" icon="mdi:database">
    Es un **ORM (Object Relational Mapper)** para Node.js.

    Permite trabajar con bases de datos usando **modelos y objetos** en lugar de escribir SQL directamente.

    Se utiliza para:

    * Definir modelos de datos
    * Crear relaciones entre tablas
    * Gestionar migraciones y consultas

    Instalación:

    ```bash
    npm install sequelize
```
  </Card>
</CardGrid>


## Dependencias de desarrollo

Estas dependencias solo se utilizan durante el desarrollo.

---

<CardGrid>
  <Card title="TypeScript" icon="logos:typescript">
    TypeScript añade **tipado estático** a JavaScript.

    Ventajas:

    * Detección temprana de errores
    * Mejor autocompletado
    * Código más mantenible

    Instalación:

    ```bash
    npm install -D typescript
    ```
  </Card>

  <Card title="@types/express" icon="logos:express">
    @types/express proporciona definiciones de tipos para usar Express con TypeScript.

    Instalación:

    ```bash
    npm install -D @types/express
    ```
  </Card>
  <Card title="@types/cors" icon="logos:cors">
    @types/cors añade tipado a la librería cors.

    Instalación:

    ```bash
    npm install -D @types/cors
    ```
  </Card>
  <Card title="@types/multer" icon="logos:multer">
    @types/cors añade tipado a la librería cors.

    @types/multer proporciona soporte de tipos para multer.

    Instalación:

    ```bash
    npm install -D @types/multer
    ```
  </Card>
  <Card title="TSLint" icon="logos:TSLint">
    TSLint es una herramienta de análisis estático para revisar la calidad del código TypeScript.

    Permite detectar:

    * Errores comunes
    * Problemas de estilo
    * Malas prácticas

    Instalación:

    ```bash
    npm install -D tslint
    ```
  </Card>

</CardGrid>


## Script de desarrollo

<CardGrid>
  <Card title="TSLint" icon="logos:dev">
    
    Pasar de Typescript a JavaScript

    ```bash
    tsc -watch
    ```      

    El proyecto incluye el siguiente script en `package.json`:

    ```json
    "scripts": {
      "dev": "node --watch dist/app.js"
    }
    ```

    Este script ejecuta el servidor y **reinicia automáticamente cuando cambia el código compilado**.

    Para iniciarlo:

    ```bash
    npm run dev
    ```
  </Card>
</CardGrid>



## Resumen

El backend del proyecto utiliza:

* Node.js
* Express
* TypeScript
* Sequelize
* MySQL
* Multer
* Sharp

Este conjunto permite construir una **API robusta capaz de gestionar formularios, imágenes y datos almacenados en base de datos**.
